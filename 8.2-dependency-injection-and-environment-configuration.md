## Purpose and Scope
The purpose of this module is to provide dependency injection and environment configuration for the application. This module fits into the overall architecture by enabling the application to dynamically configure its dependencies based on the environment it's running in.

The design decisions made in this module include using Microsoft.Extensions.DependencyInjection for dependency injection, and environment-specific configurations for database connections and other services. The benefits of this design are improved flexibility and maintainability, as well as better support for different environments (e.g., development, testing, production).

### Configuration Flow
#### Environment Detection

```csharp
public static void AddDatabaseContexts(this IServiceCollection services, IWebHostEnvironment environment, ConfigurationManager configuration)
{
    if (environment.IsDevelopment() || environment.IsDocker())
    {
        // Configure SQL Server (local)
        services.ConfigureLocalDatabaseContexts(configuration);
    }
    else
    {
        // Configure SQL Server (prod)
        var credential = new ChainedTokenCredential(new AzureDeveloperCliCredential(), new DefaultAzureCredential());
        configuration.AddAzureKeyVault(new Uri(configuration["AZURE_KEY_VAULT_ENDPOINT"] ?? ""), credential);

        services.AddDbContext<CatalogContext>((provider, options) =>
        {
            var connectionString = configuration[configuration["AZURE_SQL_CATALOG_CONNECTION_STRING_KEY"] ?? ""];
            options.UseSqlServer(connectionString, sqlOptions => sqlOptions.EnableRetryOnFailure())
                .AddInterceptors(provider.GetRequiredService<DbCallCountingInterceptor>());
        });

        {
            var connectionString = configuration[configuration["AZURE_SQL_IDENTITY_CONNECTION_STRING_KEY"] ?? ""];
```
