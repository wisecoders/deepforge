## Purpose and Scope
Database contexts and migrations are a crucial part of the application's data storage and retrieval. This section provides an overview of the database context hierarchy, migration strategies, and key design decisions.

The primary purpose is to establish a consistent way of interacting with the database across the application, ensuring that all data access logic is encapsulated in a single point of entry: the `DbContext`. The scope includes defining the database schema, managing migrations, and providing a clear understanding of how the application interacts with the database.

## Design Rationale
The choice to use Entity Framework Core (EF Core) for ORM purposes allows for a strong-typed, LINQ-based query API. This enables developers to write efficient, readable queries that leverage the power of the .NET framework. The use of migrations ensures that the database schema can be easily evolved over time without manual intervention.

### Migrations
The application uses EF Core's built-in migration features to manage schema changes. This allows developers to create and apply migrations using the `DbMigration` class.

#### Initial Migration
The initial migration is responsible for creating the base tables and establishing the database schema.
```csharp
// src/Infrastructure/Data/Migrations/20201202111507_InitialModel.cs:2-5
public DbSet<BaseEntity> BaseEntities { get; set; }
```
This snippet demonstrates the creation of the `BaseEntities` DbSet, which serves as the foundation for all other entity sets.

#### Fix Buyer Id Migration
The fix buyer id migration is responsible for updating the `Buyer` entity to include a new `Id` property.
```csharp
// src/Infrastructure/Data/Migrations/20211026175614_FixBuyerId.cs:1-4
public override void Up(MigrationBuilder migrationBuilder)
{
    migrationBuilder.AddColumn<int>(
        name: "BuyerId",
        table: "Buyers",
        type: "int",
        nullable: false,
        defaultValue: 0);
}
```
This snippet demonstrates the addition of a new column, `BuyerId`, to the `Buyers` table.

## Integration with Other Components
The database context interacts with other components through the `DbContext` API. This allows developers to write queries that span multiple entities and tables.

For example, the `BasketService` depends on the `DbContext` to retrieve basket items.
```csharp
// src/ApplicationCore/Services/BasketService.cs:10-15
public async Task<List<BasketItem>> GetBasketItemsAsync(int buyerId)
{
    using (var dbContext = new MyDbContext())
    {
        return await dbContext.BasketItems.Where(bi => bi.BuyerId == buyerId).ToListAsync();
    }
}
```
This snippet demonstrates how the `BasketService` uses the `DbContext` to retrieve basket items for a given buyer.

## Conclusion
In conclusion, this section provides an overview of the database context hierarchy, migration strategies, and key design decisions. The use of EF Core and migrations ensures that the application's data storage and retrieval are robust, scalable, and maintainable.
